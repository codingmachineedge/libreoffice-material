Tenth Material VCL milestone

Disabled affordances now read as disabled.

An audit of VCL's real native-draw calls found three controls whose disabled state collapsed onto a generic one. A disabled submenu arrow now dims, a disabled-but-checked toolbar button keeps a dimmed check, and a disabled tab control still marks its current page. The XML changes have not been compiled, launched, or captured as LibreOffice; the accepted screenshot count remains zero.

Source present Build pending
01

Dimmed submenu arrow

A disabled submenu parent passes ControlState::NONE; the arrow now strokes @outline so it dims with the rest of the disabled item.

02

Disabled, still checked

A checked toolbar button that is disabled keeps a dimmed @outline checked outline instead of collapsing to the plain disabled fill.

03

Disabled tab keeps its page

When the whole tab control is disabled, an enabled="false" selected="true" state keeps the current page identifiable rather than blank.

04

Audit-driven, scoped

Found by cross-checking VCL draw calls; three further verified gaps were left out as design decisions, not corrections. No new tokens; all affected source remains unexecuted.

Future runtime opt-in

Both environment gates are required.

The Windows non-printer backend reaches the file-widget path only when the existing draw-from-file gate is present; the second value selects the packaged Material definition. 

VCL_DRAW_WIDGETS_FROM_FILE=1
VCL_FILE_WIDGET_THEME=material

Current build gate: no complete supported profile exists. WSL has no distribution; selectable VS 2022 lacks ATL and its configured CMake; and the registry selects incomplete SDK 28000 over usable SDK 26100. The C++ targets and real app run remain pending.

Design direction

Material hierarchy without losing desktop power.

The goal is a coherent interaction system, not a layer of rounded corners. Shared primitives come first, then common chrome, then the specialized workflows professionals depend on.

01

Semantic by default

Color, type, shape, metric, elevation, state, density, and motion resolve from shared roles instead of application-local constants.

Tokens → VCL → suite
02

Dense when it matters

Compact and comfortable profiles preserve Calc throughput, keyboard fluency, touch access, and high-zoom legibility.

Adaptive, not diluted
03

Accessible at the core

Focus, roles, high contrast, reduced motion, bidirectional layout, and long translations belong in every component contract.

No screenshot-only approvals

Interactive design reference

See the target design, end to end.

A hand-built, dependency-free HTML rendering of the Material Design 3 suite—Start Center, Writer, Calc, Impress, Draw, Base, Math, a component gallery, the full 2,433-command catalog, version history, and shared dialogs. Switch light, dark, and high-contrast themes, compact and comfortable density, and classic or ribbon chrome to preview how the semantic tokens resolve. Every search bar carries a full regex builder—token palette, i/g/m/s flags, live validity and match count—filtering real data, and the Dialogs surface includes a working Find & Replace that runs the same builder over live document text.

Headless proof loop

A real desktop. Off screen. Fully attributable.

The low-level driver has already passed a Notepad-only off-screen create, enumerate, capture, and cleanup preflight. Once LibreOffice can be built, the same mechanics will target the real binary and record attributable checkpoints without disturbing the operator’s session.

Harness proof, not UI proof: driver commit 806d9ba85e4a… passed on 2026-07-16. Its temporary Notepad capture was discarded and the accepted Material evidence count remains zero.
