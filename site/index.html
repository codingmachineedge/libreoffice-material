Tenth Material VCL milestone

Disabled affordances now read as disabled.

An audit of VCL's real native-draw calls found three controls whose disabled state collapsed onto a generic one. A disabled submenu arrow now dims, a disabled-but-checked toolbar button keeps a dimmed check, and a disabled tab control still marks its current page. The exact source has now compiled and launched, while these three specific disabled-state tuples have not yet been visually exercised.

Windows updater source

Strict release metadata, protected staging, explicit consent.

The Windows-only updater reads the exact GitHub Latest XML asset and accepts one tag-derived repository URL only when the canonical MSI name, application/x-msi MIME, positive size, and lowercase SHA-256 all match. Legacy state is rejected. Confirmed bytes are copied with CREATE_NEW into protected LocalAppData staging with a user/Administrators/SYSTEM DACL, verified again, and held under a final read lock before a visible Windows Installer launch. Current source uses the exact major- update vector: /i, the staged MSI, REBOOT=ReallySuppress, and MSIRESTARTMANAGERCONTROL=DisableShutdown. Repair-only REINSTALL properties are excluded. Runtime proof of MSI restart suppression still requires lifecycle testing.

Confirmation defaults to No; no silent install exists. Checks default on weekly, automatic download defaults off, and download and install remain opt-in. See the privacy details.

The off-screen verification source also includes a bounded, read-only UNO accessibility-tree collector. With a matching built Python runtime it records roles, names, states, and bounds alongside a screenshot without reading document text or driving the UI. Corrected light, dark, and forced-high-contrast Start Center runs exercised it nine times with no collector errors; every profile exposes the focused Open File button. This is not a full accessibility audit.

Native targets, a corrected local exact-source MSI, corrected light, dark, and high-contrast Start Center headless/UNO-tree smoke, and a normal public corrected Latest release have completed. Design direction

Material hierarchy without losing desktop power.

The goal is a coherent interaction system, not a layer of rounded corners. Shared primitives come first, then common chrome, then the specialized workflows professionals depend on.

01

Semantic by default

Color, type, shape, metric, elevation, state, density, and motion resolve from shared roles instead of application-local constants.

Tokens → VCL → suite
02

Dense when it matters

Compact and comfortable profiles preserve Calc throughput, keyboard fluency, touch access, and high-zoom legibility.

Adaptive, not diluted
03

Accessible at the core

Focus, roles, high contrast, reduced motion, bidirectional layout, and long translations belong in every component contract.

No screenshot-only approvals

One system, every workspace

A whole-suite commitment.

Application workspaces remain planned while shared VCL and Start Center source work is underway. None is complete.
